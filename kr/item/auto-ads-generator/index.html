RSS 데이터 서버 API 시스템

나만의 RSS 데이터 서버 API를 생성해서, 간편하게 외부 사이트/블로그의 최신 포스트 업데이트 정보를 사용해 보세요.

1. 본인 또는 타인이 운영하고 있는 외부 사이트/블로그의 최신 글 목록 데이터를 손쉽게 가져와서, 원하는 곳에 자유롭게 활용할 수 있습니다. ( 외부 RSS 데이터 -> 내 RSS 데이터 서버 -> 원하는 곳에 불러 온 데이터 표시 )

2. 운영 중인 사이트(워드프레스 등)의 루트 디렉토리(Public, Root Directory)에 새로운 폴더를 생성한 뒤 다운로드한 파일(Zip)을 압축 해제 후 업로드하면, 별다른 설정 과정 없이 즉시 사용 가능합니다. ( 예: www.OOO.com/api/{ 업로드 } )

3. 네이버 블로그 / 티스토리 / 유튜브 / 브런치를 지원합니다. (계속 업데이트)

데모 사이트 및 사용 방법

RSS API 데이터 서버 : 데모 보기

데모용 웹페이지를 통해 외부 RSS 데이터를 가져와서 나만의 RSS API로 생성된 모습을 확인할 수 있습니다.

기본적으로 설정되어 있는 '워드프레스(WordPress)' 외에 '네이버 블로그(Naver Blog)', '티스토리(Tistory)', '유튜브(YouTube)', '브런치(Brunch)'의 데이터를 가져올 수 있습니다.

기본 URL 구조

예시에서 사용된 이름 외에도, 서비스( 'wordpress', 'youtube' 등 )에 따라 원하는 폴더명으로 자유롭게 설정할 수 있습니다. (가급적 예시와 같이 사용 권장)

- 워드프레스(WordPress) : .../api/wordPress/

- 유튜브(YouTube) : .../api/youtube/

- 네이버 블로그(Naver Blog) : .../api/naver-blog/

- 티스토리(Tistory) : .../api/tistory/

- 브런치(Brunch) : .../api/brunch/

외부 사이트/블로그 URL 설정 방법

불러 오고자 하는 외부 사이트/블로그의 URL을 설정하고자 할 경우, 'feed' 파라미터 키에 해당 사이트/블로그의 RSS 주소를 입력합니다.

- 워드프레스(WordPress) : .../api/wordPress/?feed=https://www.woocommerce.com/feed

- 유튜브(YouTube) : .../api/youtube/?feed=https://www.youtube.com/feeds/videos.xml?channel_id=UCnaGfDaiM-pM1l-SXYpq37Q

- 네이버 블로그(Naver Blog) : .../api/naver-blog/?feed=https://rss.blog.naver.com/mkttalk.xml

- 티스토리(Tistory) : .../api/tistory/?feed=https://news.mkttalk.com/rss

- 브런치(Brunch) : .../api/brunch/?feed=https://brunch.co.kr/rss/@@2t6e

[참고] 주요 플랫폼 RSS 피드 주소(URL) 간편 확인 방법 : 자세히 보기

외부 사이트/블로그 최신 글 갯수 설정 방법

불러 오고자 하는 데이터의 갯수를 설정하고자 할 경우, 'count' 파라미터 키에 갯수(숫자)를 입력합니다.

- 워드프레스(WordPress) : .../api/wordPress/?count=3

- 유튜브(YouTube) : .../api/youtube/?count=3

- 네이버 블로그(Naver Blog) : .../api/naver-blog/?count=3

- 티스토리(Tistory) : .../api/tistory/?count=3

- 브런치(Brunch) : .../api/brunch/?count=3

기타 활용 방법

외부 사이트/블로그 URL과 불러올 데이터 갯수를 함께 설정하고자 할 경우, 아래와 같이 입력할 수 있습니다.

- 워드프레스(WordPress) : .../api/wordPress/?feed=https://www.woocommerce.com/feed&count=3

운영 중인 사이트에서 생성한 RSS API 데이터는 자바스크립트 또는 기타 언어(PHP, Python 등)를 통해 해당 데이터를 활용할 수 있습니다.

데모용 웹페이지는 실제 데이터 서버로 작동하는데 필요한 기능이 일부 제한되어 있습니다.
